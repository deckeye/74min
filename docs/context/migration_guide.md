# プロジェクト運用ガイド: Playground vs Workspace

## Q1: PlaygroundからWorkspaceへ移動すべき？
**A: はい、お勧めします。**

現在の「74min」は、単なる実験（Playground）の枠を超えています。
- ✅ **技術検証完了**: 静的サイト + Supabase構成で動くことがわかった
- ✅ **外部連携済み**: GitHub, Vercel, Supabaseと接続されている
- ✅ **バージョン管理中**: Gitで履歴が管理されている

このように「育てていくプロジェクト」になった段階で、恒久的な場所（Workspace）へ移動させるのがベストプラクティスです。

## Q2: PlaygroundとWorkspaceの使い分けは？
現状の使い方は**完璧**です。

1.  **Playground（砂場）**:
    -   「これ作れるかな？」と思い立った時にとりあえず作る場所。
    -   失敗したらフォルダごと消せばいい。
    -   今回はここでスタートアップし、成功しました。

2.  **Workspace（仕事場/主要プロジェクト）**:
    -   長期的にメンテナンスする場所。
    -   `Documents/Projects/` など、バックアップ対象のフォルダに置くのが一般的。

## 推奨される移行手順 (Git Clone方式)
ファイルを直接「移動」するのではなく、GitHub経由で「複製」するのが最も安全でスマートです。

1.  **準備**:
    -   現在の変更がすべてGitHubにプッシュされていることを確認（`git push origin main`）。

2.  **新しい場所へ**:
    -   PCのエクスプローラーで、プロジェクトを置きたい場所（例: `C:\Users\user\Projects`）を開く。
    -   ターミナルで以下を実行：
        ```bash
        git clone https://github.com/deckeye/74min.git
        ```

3.  **環境切り替え**:
    -   新しいフォルダ (`74min`) をエディタ（VS Code等）で開く。
    -   Antigravity（私）も新しい場所で起動する。

この方法なら、万が一移行に失敗しても元のPlaygroundのデータは無傷なので安心です。
