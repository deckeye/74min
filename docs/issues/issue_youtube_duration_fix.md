# Issue: YouTube再生時間取得の不具合修正 (#24)

## 背景
YouTubeからの検索結果において、1曲目（あるいは特定の動画）の再生時間が正しく取得できない、または表示が0:00になる現象が報告された。調査の結果、YouTube APIのレスポンス処理におけるエラーハンドリングの不足と、不適切なデータ加工（無効なIDの連結）が原因であることが判明した。

## 目的
YouTube API連携の堅牢性を向上させ、安定して再生時間を取得・表示できるようにする。

## 変更内容
- `js/app.js`:
    - `searchYouTube` 関数におけるAPIレスポンスのステータスチェック追加。
    - 検索結果から無効なID（videoIdがないもの）を除外。
    - `fetchTrackDetails` 関数におけるガード節の追加とエラーログ出力。
    - HTMLエンティティ（`&amp;`等）のデコード処理を追加。

## 設計判断
- **HTMLデコードの統一**: `textarea` 要素を利用した標準的な手法でデコードを行うヘルパー関数 `decodeHtmlEntities` を追加した。
- **フィルタリング**: 動画IDがない項目を `fetchTrackDetails` に渡さないようにすることで、APIリクエストの失敗を未然に防ぐ。

## 完了条件
- [x] YouTube検索結果に再生時間が表示される。
- [x] 特殊文字を含む曲名が正しく表示される。
- [x] 1曲目が異常なデータであっても2曲目以降に影響しない。
