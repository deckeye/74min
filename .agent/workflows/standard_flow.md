# AIエージェント標準ワークフロー (AI Agent Standard Flow)

本プロジェクトに従事するすべてのAIエージェントは、作業開始時に必ずこのチェックリストを確認し、各ステップを順守してください。

## 1. 準備フェーズ (Preparation)
- [ ] **Issueの確認/作成**: 
    - 現在のタスクに対応する GitHub Issue が存在するか確認する。
    - 存在しない場合は、テンプレートに従って新規作成する。
- [ ] **実装プランの作成**: 
    - `implementation_plan.md` を作成（または更新）し、ユーザーの承認を得る。
    - 日本語で記述すること。

## 2. 実装フェーズ (Implementation)
- [ ] **コミットメッセージ**: 
    - コミットメッセージには必ず Issue 番号を含める（例: `fix: #19 ...`）。
- [ ] **設計判断の記録**: 
    - レイアウト、技術選定、ラベル名などの「選択」を行った場合、その理由を Issue のコメントに即座に残す。
    - 「なぜそれを、他の選択肢と比較して選んだか」を重視する。

## 3. 完了フェーズ (Completion)
- [ ] **事後ドキュメント**: 
    - `walkthrough.md` を更新し、実施内容と検証結果をまとめる。
- [ ] **Issueのクローズ**: 
    - すべての作業が完了し、ユーザーの確認が得られたら Issue をコメント付きでクローズする。
    - コメントには「何が解決したか」と「関連コミット」を含める。
- [ ] **セルフ監査**: 
    - 本セッションで作成したファイルがすべて git add/commit されているか、`git status` で確認する。
    - `docs/issues/` 下に作成したドキュメントが Git に含まれているか特に注意する。
